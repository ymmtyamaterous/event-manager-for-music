### 3.2 ログインページ（`/login`）

**URL**: `/login`
**ファイル**: `src/app/login/page.tsx`
**アクセス制限**: なし

#### フォーム項目

| 項目名 | 入力タイプ | 必須 | バリデーション |
|--------|-----------|------|---------------|
| メールアドレス | `email` | ○ | HTML5 標準 |
| パスワード | `password` | ○ | 必須チェック |

#### 表示要素
- エラーメッセージ（認証失敗時: 「メールアドレスまたはパスワードが正しくありません」）
- 「新規登録はこちら」リンク → `/register`

#### 動作
- 成功時: APIからJWTを取得して保存 → トップページ（`/`）へ遷移
- 失敗時: エラーメッセージ表示
- ボタン押下中: スピナー表示（`disabled` + ローディングスピナー）

### 3.3 新規登録ページ（`/register`）

**URL**: `/register`
**ファイル**: `src/app/register/page.tsx`
**アクセス制限**: なし

#### フォーム項目

| 項目名 | 入力タイプ | 必須 | 備考 |
|--------|-----------|------|------|
| ユーザー種別 | radio | ○ | ライブ運営/出演者/観客 |
| 姓 | text | ○ | placeholder: 山田 |
| 名 | text | ○ | placeholder: 太郎 |
| 表示用ニックネーム | text | ○ | 他ユーザーへの表示名 |
| メールアドレス | email | ○ | |
| パスワード | password | ○ | 最低8文字、英数字混在 |
| パスワード（確認） | password | ○ | 一致チェック |
| 利用規約同意 | checkbox | ○ | 未チェック時は登録ボタン無効 |

#### バリデーション
- パスワードと確認パスワードが一致しない場合: フィールド下エラー表示
- パスワード強度不足: フィールド下エラー表示
- メールアドレス重複: APIエラーをフォーム上に表示

#### 動作
- 成功時: JWTを取得して保存 → トップページ（`/`）へ遷移

- 投稿日

## 4. レスポンシブ対応

全画面でレスポンシブデザインを採用。TailwindCSS のブレークポイントに従う。

| ブレークポイント | 対応 |
|----------------|------|
| デフォルト（モバイル） | 1カラムレイアウト、ボタン全幅（`w-full`） |
| `md`（768px〜） | 2〜3カラムレイアウト |
| `lg`（1024px〜） | 2カラム（サイドバー付き）レイアウト |

---

## 5. 共通UI要件

| 要件 | 実装 |
|------|------|
| ロード状態 | ボタンを `disabled` にしてスピナー表示（`animate-spin`） |
| エラー表示 | `bg-red-50 border border-red-200 text-red-700` のインラインアラート |
| 成功表示 | `bg-green-50 border border-green-200 text-green-700` のインラインアラート |
| 確認ダイアログ | カスタムモーダルコンポーネント（`bg-black/50` オーバーレイ） |
| ホバーエフェクト | カードに `hover:shadow-md transition-shadow` |
| ページ遷移 | Next.js `<Link>` コンポーネント、または `router.push()` |
| データ未取得時 | `null` / `undefined` チェック後に表示（Optional chaining使用）|

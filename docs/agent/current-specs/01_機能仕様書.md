# 機能仕様書

> **作成日**: 2026-02-25
> **対象バージョン**: Event Manager v1.0

---

## 1. システム概要

**EventManager** は、ライブ音楽イベントを中心とした管理・予約プラットフォームです。

| レイヤー | 技術スタック |
|---------|------------|
| フロントエンド | Next.js (App Router) + TypeScript + TailwindCSS |
| バックエンド | Go + Air（ホットリロード） |
| データベース | PostgreSQL |
| 認証 | JWT（Bearer Token） |
| ファイルアップロード | サーバーサイド保存（環境変数 `UPLOAD_DIR` 配下） |

---

## 2. ユーザー種別

| 種別 | 識別子 | 説明 |
|------|--------|------|
| 管理者 | `admin` | システム全体を管理する |
| 運営者 | `organizer` | ライブイベントを企画・運営する |
| 出演者 | `performer` | バンドを登録してイベントへ出演申請を行う |
| 観客 | `audience` | イベントを閲覧・予約する |

## 3. 機能一覧

### 3.1 認証機能

| 機能ID | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| AUTH-01 | ログイン | メールアドレス + パスワードによるログイン（JWT発行） | 全ユーザー |
| AUTH-02 | 新規登録 | ユーザー種別・名前・メールアドレス・パスワードを登録 | 全ユーザー |
| AUTH-03 | ログアウト | クライアント側のJWTを破棄しトップページへ遷移 | ログイン済みユーザー |
| AUTH-04 | トークンリフレッシュ | アクセストークンをリフレッシュトークンで更新 | ログイン済みユーザー |

#### AUTH-01 詳細：ログイン
- 入力項目: メールアドレス、パスワード
- バックエンド処理:
  - DBからメールアドレスでユーザーを検索
  - bcrypt でパスワードを検証
  - 検証成功時: アクセストークン（1時間）＋リフレッシュトークン（30日）を発行
- レスポンス: `{ access_token, refresh_token, user }`
- トークン保管: フロントエンドで `localStorage` または `httpOnly Cookie` に保存

#### AUTH-02 詳細：新規登録
- 入力項目: ユーザー種別（ラジオボタン）、姓、名、表示用ニックネーム、メールアドレス、パスワード、パスワード（確認）、利用規約同意
- バリデーション:
  - メールアドレスの重複チェック（サーバーサイド）
  - パスワードと確認パスワードの一致チェック（フロントエンド・サーバーサイド両方）
  - パスワード: 最低8文字、英数字混在
  - 利用規約への同意が必須
- パスワード保存: bcrypt によるハッシュ化
- 登録後: JWTを発行してログイン状態でトップページへ遷移

### 3.2 プロフィール機能

| 機能ID | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| PROF-01 | プロフィール閲覧 | 自分の登録情報を表示 | ログイン済みユーザー |

- 保存方式: バックエンド API 経由でサーバーの `UPLOAD_DIR` に保存
- レスポンス: 保存されたファイルのパス（URLとして返却）

### 3.3 イベント閲覧機能（一般公開）

| 機能ID | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| EVT-01 | イベント一覧 | 公開中のイベントを一覧表示 | 全ユーザー（未ログインも可） |
| EVT-02 | イベント検索 | イベント名・会場名・説明文でフィルタリング | 全ユーザー |
| EVT-03 | イベント詳細 | イベントの詳細情報・タイムテーブル・お知らせを表示 | 全ユーザー |

#### EVT-01 詳細：イベント一覧
- ステータスが `published` のイベントのみ表示
- 表示情報: タイトル・開催日時・会場名・説明（一部）・チケット料金・定員

#### EVT-03 詳細：イベント詳細
- 表示情報:
  - 基本情報（開催日、時間、会場名・住所、料金、定員/予約数）
  - タイムテーブル（出演バンド一覧、出演時間）
  - お知らせ一覧
  - SNS リンク（Twitter/X, Instagram, Facebook等）
  - 予約ボタン（観客ユーザーのみ）

### 3.4 予約機能

| 機能ID | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| RES-01 | イベント予約 | イベント詳細ページから予約を作成 | 観客 |
| RES-02 | 予約一覧表示 | 自分の予約一覧（有効/キャンセル済み）を表示 | 観客 |
| RES-03 | 予約キャンセル | 予約中のイベントをキャンセル | 観客 |

#### RES-01 詳細：イベント予約
- 前提条件:
  - ログイン済み かつ `user_type = audience`
  - 定員に達していない（`capacity` が設定されている場合）
  - 同一イベントへの重複予約なし（`status = reserved` の予約が存在しない）
- 予約番号: バックエンドで `RES{timestamp}` 形式を自動生成
- 予約完了後: 観客ダッシュボードへ遷移

### 3.5 運営者機能

| 機能ID | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| ORG-01 | 運営ダッシュボード | 自分のイベント一覧（公開中/下書き）と統計を表示 | 運営者 |
| ORG-02 | イベント作成 | 新規イベントを下書きまたは公開状態で作成 | 運営者 |
| ORG-03 | イベント編集 | 既存イベントの情報を変更 | 運営者（自分のイベントのみ） |
| ORG-04 | イベント削除 | イベントを削除（確認ダイアログあり） | 運営者（自分のイベントのみ） |
| ORG-05 | 出演バンド管理 | タイムテーブルへのバンド直接追加・削除 | 運営者 |
| ORG-06 | エントリー申請管理 | 出演申請の承認・却下 | 運営者 |
| ORG-07 | お知らせ管理 | お知らせの作成・編集・削除 | 運営者 |
| ORG-08 | 予約者一覧 | 予約者リストの閲覧・CSV出力 | 運営者 |

#### ORG-02 詳細：イベント作成
- 入力項目:
  - **必須**: タイトル、会場名、会場住所、開催日、開場時間、開演時間、公開ステータス
  - **任意**: 説明、終演時間、チケット料金、定員、予約受付期間、SNSリンク（Twitter/X, Instagram, Facebook, その他）

#### ORG-06 詳細：エントリー申請管理
- ステータス別（未対応/承認済み/却下）に表示
- 承認時: `performances` テーブルにレコードを自動追加
- 却下時: 却下理由の入力が必須

#### ORG-08 詳細：予約者一覧
- 表示情報: 予約番号、予約者名、メールアドレス、ステータス、予約日時
- 検索: 予約番号・表示名・メールアドレスで検索
- フィルター: 全件/予約中/キャンセル済み
- エクスポート: CSV形式でダウンロード（UTF-8 BOM付き）

### 3.6 出演者機能

| 機能ID | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| PFM-01 | 出演者ダッシュボード | 自分のバンド一覧・出演予定を表示 | 出演者 |
| PFM-02 | バンド登録 | 新規バンド情報を登録 | 出演者 |
| PFM-03 | バンド編集 | バンド情報・メンバー情報を編集 | 出演者（自分のバンドのみ） |
| PFM-04 | バンド詳細閲覧 | バンドプロフィールページ（公開） | 全ユーザー |
| PFM-05 | イベントエントリー一覧 | エントリー可能なイベント一覧と申請状況を表示 | 出演者 |
| PFM-06 | エントリー申請 | イベントへの出演申請を送信 | 出演者 |

#### PFM-02 詳細：バンド登録
- 入力項目:
  - **必須**: バンド名
  - **任意**: ジャンル、結成年、説明、プロフィール画像（5MB以下）、SNSリンク、メンバー情報（名前・担当パート）
- プロフィール画像: バックエンド API 経由でサーバーの `UPLOAD_DIR` に保存
- メンバー情報: 動的に追加・削除可能

#### PFM-06 詳細：エントリー申請
- 申請可能条件: バンドを1つ以上登録済み、かつ同一バンドで重複申請なし
- バンド選択: 自分が保有するバンドから選択（既申請バンドは非表示）
- 任意メッセージの送信が可能
- 申請後: 出演者のイベント一覧ページへ遷移（2秒後）

## 4. アクセス制御

| ページ | 未ログイン | 観客 | 出演者 | 運営者 | 管理者 |
|--------|-----------|------|--------|--------|--------|
| トップページ | ○ | ○ | ○ | ○ | ○ |
| イベント一覧 | ○ | ○ | ○ | ○ | ○ |
| イベント詳細 | ○（予約不可） | ○ | ○ | ○ | ○ |
| バンド詳細 | ○ | ○ | ○ | ○ | ○ |
| 観客ダッシュボード | ✗（→ログイン） | ○ | ✗（→TOP） | ✗（→TOP） | ✗（→TOP） |
| 運営ダッシュボード | ✗（→ログイン） | ✗（→TOP） | ✗（→TOP） | ○ | ✗（→TOP） |
| 出演者ダッシュボード | ✗（→ログイン） | ✗（→TOP） | ○ | ✗（→TOP） | ✗（→TOP） |
| プロフィール | ✗（→ログイン） | ○ | ○ | ○ | ○ |

### 実装方針
- フロントエンド: Next.js Middleware または各ページコンポーネントでJWTのペイロードを確認してリダイレクト
- バックエンド: Go の Middleware でJWT署名を検証し、`user_type` によるロールチェックを実施

## 5. ファイルアップロード仕様

| 項目 | 仕様 |
|------|------|
| 対象 | プロフィール画像、バンドプロフィール画像 |
| 対応形式 | `image/jpeg`, `image/png`, `image/gif`, `image/webp` |
| 最大サイズ | 5MB |
| 保存先 | 環境変数 `UPLOAD_DIR` で指定したディレクトリ |
| ファイル名 | UUIDによるリネーム（衝突防止） |
| 配信 | バックエンド静的ファイル配信 または CDN |

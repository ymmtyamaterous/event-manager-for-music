## 3. セキュリティ要件

### 3.1 認証・認可

| 項目 | 要件 |
|------|------|
| パスワード | bcrypt でハッシュ化（cost factor: 10以上）、最低8文字、英数字混在 |
| JWT | HS256署名、アクセストークン1時間、リフレッシュトークン30日 |
| JWT秘密鍵 | 環境変数 `JWT_SECRET`（32文字以上のランダム文字列） |
| HTTPS | 全通信をHTTPS化（TLS 1.2以上） |
| CORS | 環境変数 `ALLOWED_ORIGINS` で許可オリジンを限定 |
| CSRFト対策 | JWTをAuthorizationヘッダーで送信（Cookie不使用時は不要） |
| XSS対策 | React の JSX による自動エスケープ + Content Security Policy ヘッダー |
| 入力検証 | Go バックエンドでサーバーサイドバリデーション必須 |
| SQLインジェクション対策 | プリペアドステートメント（Go標準 `database/sql` のプレースホルダー） |

### 3.2 アクセス制御

| 項目 | 要件 |
|------|------|
| ロールベースアクセス制御 | `admin` / `organizer` / `performer` / `audience` の4種別 |
| リソースオーナーシップ | イベントは作成者のみ編集可、バンドはオーナーのみ編集可 |
| APIレート制限 | 1分間あたり60リクエスト/ユーザー（将来実装） |

### 3.3 データ保護

| 項目 | 要件 |
|------|------|
| パスワード | DBにハッシュ値のみ保存（平文保存禁止） |
| 個人情報 | ログに個人情報（メールアドレス等）を含めない |
| アップロード画像 | `UPLOAD_DIR` 配下にUUIDリネームして保存 |

## 4. 保守性・拡張性要件

### 4.1 フロントエンド

| 項目 | 要件 |
|------|------|
| 言語 | TypeScript（strict モード有効） |
| フレームワーク | Next.js（App Router） |
| スタイリング | TailwindCSS |
| Linter / Formatter | ESLint + Prettier |
| 型定義 | `src/types/index.ts` に集約（`any` 型使用禁止） |
| null/undefined 安全性 | API取得データはOptional chainingで安全にアクセス |
| コンポーネント分割 | Feature-based または Atomic Design |

### 4.2 バックエンド

| 項目 | 要件 |
|------|------|
| 言語 | Go |
| ホットリロード | Air（開発時） |
| DBアクセス | `database/sql` またはクエリビルダー（sqlx等） |
| API定義 | `backend/openapi.yaml`（OpenAPI 3.0）で一元管理 |
| マイグレーション | Go製CLIコマンドで管理（`go run ./cmd/migrate`） |
| ログ | 構造化ログ（JSONフォーマット推奨） |

### 4.3 テスト要件

| 種別 | 要件 |
|------|------|
| フロントエンド ユニットテスト | 主要コンポーネント・ユーティリティをカバー |
| バックエンド ユニットテスト | ハンドラー・サービス層のカバレッジ 80%以上 |
| 統合テスト | 主要ユーザーフロー（予約・エントリー）をカバー |
| E2Eテスト | 予約フロー・エントリーフローを自動化（将来実装） |

## 5. UI/UX要件

### 5.1 レスポンシブデザイン

TailwindCSS のブレークポイントに従い、全画面でレスポンシブ対応する。

| ブレークポイント | 幅 | 対応 |
|----------------|-----|------|
| デフォルト（モバイル） | ～ 767px | 1カラム、全幅ボタン |
| `md` | 768px 〜 | 2〜3カラム |
| `lg` | 1024px 〜 | 2カラム（サイドバー付き） |

### 5.2 アクセシビリティ目標

| 項目 | 要件 |
|------|------|
| 準拠基準 | WCAG 2.1 AA |
| キーボード操作 | 全機能がキーボードのみで操作可能 |
| スクリーンリーダー | 意味のある `alt` テキスト・`aria` ラベル |
| カラーコントラスト | 4.5:1 以上（テキスト） |
| フォーカス表示 | TailwindCSS の `focus:ring` を使用した明確なフォーカスリング |

### 5.3 ローディング表示

| 状況 | 表示 |
|------|------|
| ボタン操作中 | ボタンを `disabled` にして `animate-spin` スピナー表示 |
| ページ初回データ取得中 | スケルトンローダー（`animate-pulse`） |
| ページ遷移 | Next.js デフォルトローディング |

### 5.4 エラーハンドリング

| 状況 | 実装 |
|------|------|
| フォームバリデーション | フィールドレベルのインラインエラーテキスト |
| APIエラー | フォーム上または画面上部のインラインアラート |
| 404エラー | `src/app/not-found.tsx` によるカスタム404ページ |
| 認証エラー（401） | `/login` へリダイレクト |
| ネットワークエラー | エラーメッセージ表示（リトライボタン付き） |
| null/undefined | Optional chaining (`?.`) で安全にアクセスし、フォールバック表示 |

## 6. 環境・インフラ要件

### 6.1 環境変数一覧

バックエンドは以下の環境変数から設定を読み込む（`.env` ファイルで管理）。

| 変数名 | 説明 | 例 |
|--------|------|----|
| `HOST` | ホスト名 | `0.0.0.0` |
| `API_PORT` | APIポート番号 | `8000` |
| `ALLOWED_ORIGINS` | CORS許可オリジン（カンマ区切り） | `http://localhost:3000` |
| `DATABASE_URL` | PostgreSQL接続URL | `postgres://user:pass@postgres:5432/dbname` |
| `JWT_SECRET` | JWT署名秘密鍵（32文字以上） | `your-secret-key-here` |
| `UPLOAD_DIR` | アップロードファイル保存先 | `/workspace/uploads` |

### 6.2 開発環境（devcontainer）

本リポジトリは devcontainer で定義されたコンテナ環境で開発する。

| コンポーネント | 構成 |
|---------------|------|
| 開発コンテナ | Ubuntu 24.04、`vscode` ユーザー |
| フロントエンド | Next.js（port: 3000） |
| バックエンド | Go + Air ホットリロード（port: 8000） |
| データベース | PostgreSQL 18（`infra/compose.yml` で定義） |
| DB管理 | pgAdmin 4（port: 5050） |

### 6.3 デプロイ環境（将来）

## 3. テーブル定義

### 3.1 users（ユーザー）

```sql
CREATE TABLE users (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email         VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    first_name    VARCHAR(100) NOT NULL,
    last_name     VARCHAR(100) NOT NULL,
    display_name  VARCHAR(100) NOT NULL,
    user_type     user_type   NOT NULL,
    profile_image_path VARCHAR(500),
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_email ON users(email);
```

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| `id` | UUID | NOT NULL | プライマリキー（自動生成） |
| `email` | VARCHAR(255) | NOT NULL | メールアドレス（ユニーク） |
| `password_hash` | VARCHAR(255) | NOT NULL | bcryptハッシュ化済みパスワード |
| `first_name` | VARCHAR(100) | NOT NULL | 姓 |
| `last_name` | VARCHAR(100) | NOT NULL | 名 |
| `display_name` | VARCHAR(100) | NOT NULL | 表示用ニックネーム |
| `user_type` | user_type | NOT NULL | ユーザー種別 |
| `profile_image_path` | VARCHAR(500) | NULL | プロフィール画像のサーバーパス |
| `created_at` | TIMESTAMPTZ | NOT NULL | 作成日時 |
| `updated_at` | TIMESTAMPTZ | NOT NULL | 更新日時 |

**user_type 定義**

| 値 | 説明 |
|----|------|
| `admin` | 管理者 |
| `organizer` | 運営者 |
| `performer` | 出演者 |
| `audience` | 観客 |

### 3.2 events（ライブイベント）

```sql
CREATE TABLE events (
    id                 UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    organizer_id       UUID         NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title              VARCHAR(255) NOT NULL,
    description        TEXT,
    flyer_image_path   VARCHAR(500),
    venue_name         VARCHAR(255) NOT NULL,
    venue_address      VARCHAR(500) NOT NULL,
    event_date         DATE         NOT NULL,
    doors_open_time    TIME         NOT NULL,
    start_time         TIME         NOT NULL,
    end_time           TIME,
    ticket_price       INTEGER,
    capacity           INTEGER,
    reservation_start  TIMESTAMPTZ,
    reservation_end    TIMESTAMPTZ,
    twitter_url        VARCHAR(500),
    instagram_url      VARCHAR(500),
    facebook_url       VARCHAR(500),
    other_url          VARCHAR(500),
    status             event_status NOT NULL DEFAULT 'draft',
    created_at         TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at         TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_events_organizer_id ON events(organizer_id);
CREATE INDEX idx_events_status        ON events(status);
CREATE INDEX idx_events_event_date    ON events(event_date);
```

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| `id` | UUID | NOT NULL | プライマリキー |
| `organizer_id` | UUID | NOT NULL | FK → users.id |
| `title` | VARCHAR(255) | NOT NULL | イベントタイトル |
| `description` | TEXT | NULL | イベント説明 |
| `flyer_image_path` | VARCHAR(500) | NULL | フライヤー画像のサーバーパス |
| `venue_name` | VARCHAR(255) | NOT NULL | 会場名 |
| `venue_address` | VARCHAR(500) | NOT NULL | 会場住所 |
| `event_date` | DATE | NOT NULL | 開催日 |
| `doors_open_time` | TIME | NOT NULL | 開場時間 |
| `start_time` | TIME | NOT NULL | 開演時間 |
| `end_time` | TIME | NULL | 終演時間 |
| `ticket_price` | INTEGER | NULL | チケット料金（円） |
| `capacity` | INTEGER | NULL | 定員（名） |
| `reservation_start` | TIMESTAMPTZ | NULL | 予約受付開始日時 |
| `reservation_end` | TIMESTAMPTZ | NULL | 予約受付終了日時 |
| `twitter_url` | VARCHAR(500) | NULL | Twitter/X URL |
| `instagram_url` | VARCHAR(500) | NULL | Instagram URL |
| `facebook_url` | VARCHAR(500) | NULL | Facebook URL |
| `other_url` | VARCHAR(500) | NULL | その他URL |
| `status` | event_status | NOT NULL | 公開ステータス（デフォルト: `draft`） |
| `created_at` | TIMESTAMPTZ | NOT NULL | 作成日時 |
| `updated_at` | TIMESTAMPTZ | NOT NULL | 更新日時 |

**event_status 定義**

| 値 | 説明 |
|----|------|
| `draft` | 下書き（一般公開されない） |
| `published` | 公開中（一般公開される） |
| `cancelled` | キャンセル済み |

### 3.5 performances（出演情報）

```sql
CREATE TABLE performances (
    id                UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    event_id          UUID        NOT NULL REFERENCES events(id) ON DELETE CASCADE,
    band_id           UUID        NOT NULL REFERENCES bands(id) ON DELETE CASCADE,
    start_time        TIME,
    end_time          TIME,
    performance_order SMALLINT    NOT NULL DEFAULT 0,
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (event_id, band_id)
);

CREATE INDEX idx_performances_event_id ON performances(event_id);
CREATE INDEX idx_performances_band_id  ON performances(band_id);
```

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| `id` | UUID | NOT NULL | プライマリキー |
| `event_id` | UUID | NOT NULL | FK → events.id |
| `band_id` | UUID | NOT NULL | FK → bands.id |
| `start_time` | TIME | NULL | 出演開始時間 |
| `end_time` | TIME | NULL | 出演終了時間 |
| `performance_order` | SMALLINT | NOT NULL | タイムテーブル上の順序 |
| `created_at` | TIMESTAMPTZ | NOT NULL | 作成日時 |
| `updated_at` | TIMESTAMPTZ | NOT NULL | 更新日時 |

**制約**: `(event_id, band_id)` の組み合わせはユニーク（同一バンドの重複出演不可）

### 3.7 reservations（予約）

```sql
CREATE TABLE reservations (
    id                 UUID               PRIMARY KEY DEFAULT gen_random_uuid(),
    event_id           UUID               NOT NULL REFERENCES events(id) ON DELETE CASCADE,
    user_id            UUID               NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    reservation_number VARCHAR(50)        NOT NULL UNIQUE,
    status             reservation_status NOT NULL DEFAULT 'reserved',
    reserved_at        TIMESTAMPTZ        NOT NULL DEFAULT NOW(),
    cancelled_at       TIMESTAMPTZ,
    created_at         TIMESTAMPTZ        NOT NULL DEFAULT NOW(),
    updated_at         TIMESTAMPTZ        NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_reservations_event_id ON reservations(event_id);
CREATE INDEX idx_reservations_user_id  ON reservations(user_id);
CREATE INDEX idx_reservations_status   ON reservations(status);

-- 同一ユーザー・同一イベントの重複予約防止（status=reservedの場合のみ）
CREATE UNIQUE INDEX idx_reservations_active
    ON reservations(event_id, user_id)
    WHERE status = 'reserved';
```

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| `id` | UUID | NOT NULL | プライマリキー |
| `event_id` | UUID | NOT NULL | FK → events.id |
| `user_id` | UUID | NOT NULL | FK → users.id（予約者） |
| `reservation_number` | VARCHAR(50) | NOT NULL | 予約番号（`RES{timestamp}`、ユニーク） |
| `status` | reservation_status | NOT NULL | 予約ステータス（デフォルト: `reserved`） |
| `reserved_at` | TIMESTAMPTZ | NOT NULL | 予約日時 |
| `cancelled_at` | TIMESTAMPTZ | NULL | キャンセル日時 |
| `created_at` | TIMESTAMPTZ | NOT NULL | 作成日時 |
| `updated_at` | TIMESTAMPTZ | NOT NULL | 更新日時 |

**reservation_status 定義**

| 値 | 説明 |
|----|------|
| `reserved` | 予約中 |
| `cancelled` | キャンセル済み |

### 3.8 entries（エントリー申請）

```sql
CREATE TABLE entries (
    id               UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    event_id         UUID         NOT NULL REFERENCES events(id) ON DELETE CASCADE,
    band_id          UUID         NOT NULL REFERENCES bands(id) ON DELETE CASCADE,
    status           entry_status NOT NULL DEFAULT 'pending',
    message          TEXT,
    rejection_reason TEXT,
    created_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    UNIQUE (event_id, band_id)
);

CREATE INDEX idx_entries_event_id ON entries(event_id);
CREATE INDEX idx_entries_band_id  ON entries(band_id);
CREATE INDEX idx_entries_status   ON entries(status);
```

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| `id` | UUID | NOT NULL | プライマリキー |
| `event_id` | UUID | NOT NULL | FK → events.id |
| `band_id` | UUID | NOT NULL | FK → bands.id |
| `status` | entry_status | NOT NULL | 申請ステータス（デフォルト: `pending`） |
| `message` | TEXT | NULL | 申請メッセージ |
| `rejection_reason` | TEXT | NULL | 却下理由 |
| `created_at` | TIMESTAMPTZ | NOT NULL | 作成日時 |
| `updated_at` | TIMESTAMPTZ | NOT NULL | 更新日時 |

**制約**: `(event_id, band_id)` の組み合わせはユニーク（同一バンドの重複申請不可）

**entry_status 定義**

| 値 | 説明 |
|----|------|
| `pending` | 申請中（未対応） |
| `approved` | 承認済み |
| `rejected` | 却下 |

## 4. マイグレーション方針

- Go製CLIコマンドでマイグレーションを実行する
- マイグレーションファイルは `backend/migrations/` ディレクトリに連番で管理

```
backend/
└── migrations/
    ├── 001_create_enums.sql
    ├── 002_create_users.sql
    ├── 003_create_events.sql
    ├── 004_create_bands.sql
    ├── 005_create_band_members.sql
    ├── 006_create_performances.sql
    ├── 007_create_announcements.sql
    ├── 008_create_reservations.sql
    └── 009_create_entries.sql
```

**実行コマンド例**:
```bash
go run ./cmd/migrate up    # マイグレーション適用
go run ./cmd/migrate down  # ロールバック
```

## 5. データ制約まとめ

| テーブル | 制約 |
|---------|------|
| `users` | `email` はユニーク |
| `events` | 同一ユーザーで複数イベント作成可 |
| `performances` | `(event_id, band_id)` はユニーク |
| `reservations` | `status = 'reserved'` の場合、`(event_id, user_id)` はユニーク（部分インデックス） |
| `reservations` | `capacity` を超えた予約はバックエンドで制御 |
| `entries` | `(event_id, band_id)` はユニーク |
| `band_members` | `display_order` による順序管理 |

## 6. セッション管理

| 項目 | 実装 |
|------|------|
| 認証方式 | JWT (Bearer Token) |
| アクセストークン有効期限 | 1時間 |
| リフレッシュトークン有効期限 | 30日 |
| 署名アルゴリズム | HS256 |
| 秘密鍵 | 環境変数 `JWT_SECRET` |
| トークン格納（フロント） | `localStorage` または `httpOnly Cookie` |
| DBへの保存 | アクセストークンはDBに保存しない（ステートレス） |
